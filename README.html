DLP Scanner / Auditor

Scan folders for exposed sensitive data and generate secure, masked audit reports.

What does this tool do?

You point it at a folder on your computer. It automatically reads every .txt and .csv file inside (including sub-folders), searches for sensitive data, and writes a clean audit report.

The report never contains the real sensitive values. It only shows a masked version so that the report itself cannot be used to steal data.

Key Features

Recursive scanningAutomatically goes through all sub-folders inside the target directory.
Smart validationUses real algorithms (Luhn, GSIS check-digit) — not just regex — to avoid false positives.
Data maskingRaw values never appear in the output. The report is safe to share.
JSON & CSV outputChoose the format that works best for your workflow or reporting tool.
Encoding-awareHandles UTF-8 and Latin-1 files. Unreadable files are skipped gracefully.
Zero dependenciesUses only the Python standard library. No extra installs needed.

Get Started in 3 Steps

1

Download the project

git clone https://github.com/YOUR_USERNAME/dlp-scanner.git
cd dlp-scanner

Or download the ZIP from GitHub and extract it.

2

Create a virtual environment (recommended)

# Create it
python -m venv venv

# Activate — Windows PowerShell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
venv\Scripts\activate

# Activate — Command Prompt (cmd)
venv\Scripts\activate.bat

# Activate — macOS / Linux
source venv/bin/activate
After activation you will see (venv) at the start of your terminal prompt.
3

Run your first scan

pip install -r requirements.txt
python main.py --target sample_data --format both --output my_first_report
Windows note: If python gives an error, use py instead.

Scanning Your Own Data

JSON report (default)

python main.py --target C:\path\to\your\folder

Both JSON and CSV at once

python main.py --target C:\path\to\your\folder --format both --output audit_july

See detailed progress while scanning

python main.py --target C:\path\to\your\folder --verbose

How Detection Works

The tool does not just match patterns. Every match is verified with a real mathematical check before being included in the report.

Credit Cards

Detects Visa, Mastercard, Amex, Discover, and Maestro — with or without spaces/dashes. Every match is then validated using the Luhn algorithm (the same checksum your bank uses). Numbers that fail are silently discarded.

Greek AFM (ΑΦΜ)

Detects exactly 9-digit numbers starting with a non-zero digit, then verifies the GSIS check-digit formula. Random 9-digit numbers that happen to appear in a document will not trigger a false positive.

Email Addresses

Uses a strict RFC 5322 pattern with word boundaries to avoid matching partial strings inside URLs or other data.

Roadmap & Future Features

The scanner is built on a plug-in detector architecture — adding a new data type requires writing a single class and registering it. No other part of the codebase needs to change. The sections below show every planned expansion, organized by category.

🔨

How to add a new detector

Every detector extends BaseDetector in dlp_scanner/detectors.py and implements one method: find_all(line). It returns a list of (raw_match, masked_value) pairs. The scanner and reporter require zero changes. This means the project can grow from 3 detectors to 30 without touching the core architecture.

Important Notes

Only scan folders you are authorized to access.
This tool is intended for internal auditing by authorized personnel. Unauthorized scanning may violate GDPR and Greek Law 4624/2019.
Report files are excluded from git.
The .gitignore ensures that generated .json and .csv reports are never accidentally committed to GitHub.
Unreadable files are skipped safely.
If a file has permission restrictions or an unsupported encoding, the scanner logs a warning and continues without crashing.
